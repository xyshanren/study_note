## 四、数据库设计

### 4.1 用户表 (users)
```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    avatar VARCHAR(255),
    bio TEXT,
    role VARCHAR(20) DEFAULT 'admin',
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 4.2 作品表 (projects)
```sql
CREATE TABLE projects (
    id SERIAL PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    description TEXT,
    tech_stack VARCHAR(500),  -- JSON 数组
    image VARCHAR(255),
    demo_url VARCHAR(255),
    source_url VARCHAR(255),
    category VARCHAR(50),
    sort_order INT DEFAULT 0,
    is_published BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 4.3 文章表 (articles)
```sql
CREATE TABLE articles (
    id SERIAL PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    slug VARCHAR(200) UNIQUE NOT NULL,
    content TEXT,  -- Markdown
    summary TEXT,
    cover_image VARCHAR(255),
    tags VARCHAR(255),  -- JSON 数组
    view_count INT DEFAULT 0,
    is_published BOOLEAN DEFAULT FALSE,
    published_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 4.4 留言表 (messages)
```sql
CREATE TABLE messages (
    id SERIAL PRIMARY KEY,
    visitor_name VARCHAR(50) NOT NULL,
    visitor_email VARCHAR(100),
    content TEXT NOT NULL,
    reply TEXT,
    is_replied BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 五、API 设计

### 5.1 作品 API
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/projects | 作品列表 |
| GET | /api/projects/{id} | 作品详情 |
| POST | /api/projects | 创建作品（管理员） |
| PUT | /api/projects/{id} | 更新作品（管理员） |
| DELETE | /api/projects/{id} | 删除作品（管理员） |

### 5.2 文章 API
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/articles | 文章列表 |
| GET | /api/articles/{slug} | 文章详情 |
| GET | /api/articles/{slug} | 增加浏览量 |

### 5.3 留言 API
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/messages | 留言列表（管理员） |
| POST | /api/messages | 提交留言 |
| PUT | /api/messages/{id}/reply | 回复留言（管理员） |
